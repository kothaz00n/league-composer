## 2024-05-14 - Optimize role lookups in getChampionStats
**Learning:** Found an O(n) array operation (`Object.keys(qData).find(k => k.toLowerCase() === role.toLowerCase())`) occurring frequently inside the hot path `getChampionStats`. The performance penalty was noticeable when evaluating hundreds of stats for draft recommendations. This happens because the keys in `qData` were not normalized to lowercase ahead of time, requiring string transformations and iterative array checking during each lookup instead of direct O(1) property access.
**Action:** Normalize role keys to lowercase during the initial data loading phase (`loadWinRates`) instead of at access time. Then modify `getChampionStats` and `getImportedChampions` to use direct dictionary lookup `qData[role.toLowerCase()]`. This transforms an O(N) operation per call into O(1), improving recommendation engine evaluation time significantly.
