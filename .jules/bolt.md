## 2024-05-24 - [Hoisting Loop-Invariant Calculations]
**Learning:** In the `getRecommendations` engine, complex O(P) operations like mapping custom archetype pools, deriving required roles (`deriveRolesFromPool`), and searching ally arrays (`allies.find()`) were being executed inside the main O(N) champion iteration loop. This caused O(N*P) complexity degradation. Pre-calculating and memoizing these values outside the loop yields a measurable performance improvement when evaluating hundreds of champions.
**Action:** Always inspect the main scoring loop in recommendation engines or any high-frequency iteration blocks for derived states or array/object lookups that depend only on inputs (e.g., target archetype definitions, roster config) and not the iterated element itself, and hoist them before the loop.
