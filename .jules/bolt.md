## 2024-04-20 - Set Membership Lookups in Nested Engine Loops
**Learning:** In the `getCompositionAnalysis` engine logic, nested iteration over all champion names occurs for each team member to find substitution suggestions. Inside this hot inner loop, an O(N) membership check using `Object.values(teamRoles).includes(...)` caused unnecessary array allocations and iterations. Pre-calculating a `Set` of current team members outside the loop structure drastically reduces redundant allocations and converts the lookup from O(N) to O(1), significantly improving the engine's performance scaling in restricted execution environments.
**Action:** Always pre-calculate Sets outside of deeply nested loops (especially O(N*P) complexity scenarios like recommendation/suggestion engines) for membership checks instead of using dynamic array derivations (`Object.values()`, `filter()`, `map()`) inside the hot path.
