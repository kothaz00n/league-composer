## 2024-05-19 - Fast Property Access vs Object.keys().find()
**Learning:** In `winRateProvider.js`, accessing role data dynamically using `Object.keys(qData).find(k => k.toLowerCase() === role.toLowerCase())` was causing significant performance overhead in the hot path of the recommendation engine. The `getChampionStats` function is called hundreds of times per draft analysis update. While the role string is usually already lowercased and exists as a direct property, `Object.keys` created temporary arrays and `find` iterated through them unnecessarily.
**Action:** Always attempt a direct property lookup (e.g., `qData[roleLower]`) before falling back to array iteration operations like `Object.keys().find()` when normalizing keys. This small change cuts the execution time of `getChampionStats` in half.
