## 2024-05-18 - [Optimize getRecommendations hot loop]
**Learning:** In highly iterated loops like the recommendation algorithm (`getRecommendations`), creating new objects using object spread (`{ ...obj }`) and calculating loop-invariant data (e.g., mapping and generating Sets from `rosterConfig` or `targetArchetypeDef`) on every iteration creates a significant garbage collection overhead and slows down the scoring function. Furthermore, missing optional chaining and null coalescing can result in multiple deep object lookups that block the main event thread, particularly when iterating over hundreds of champions.
**Action:** Extract all loop-invariant logic outside of the main loop (like deriving roles and converting arrays to Sets for O(1) lookups). In place of object spreading, use direct property lookups with nullish coalescing (`obj1[key] ?? obj2[key]`). This ensures the loop body does as little work as possible and prevents unnecessary memory allocations.
